# Goal
1. LLM을 하나의 컴포넌트/API처럼 다룰 수 있다
2. 프롬프트 구조를 이해한다
3. 웹 서비스에 연결할 수 있다
4. JSON 기반 출력 제어를 할 수 있다

# 핵심 개념
1. LLM : Stateful한 REST API + 비결정적 추론 엔진
2. Context Window : 모델이 기억할 수 있는 토큰 범위
3. Token : 텍스트의 최소 단위
4. Temperature : 창의성 (높을수록 랜덤)
5. Top-p : 확률 분포 컷
6. System Prompt : 성격, 역할
7. User Prompt : 입력
8. Assistant : 출력
